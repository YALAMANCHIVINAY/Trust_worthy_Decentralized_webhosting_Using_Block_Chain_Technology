# Documentation

This directory contains all project documentation.

## Contents

- `requirements.md` - Detailed requirements specification
- `srs.md` - Software Requirements Specification
- `architecture.md` - System architecture documentation
- `use-cases.md` - Use case descriptions
- `uml-diagrams/` - UML diagrams (use case, sequence, class, activity)
- `project-report.docx` - Complete project report
- `screenshots/` - Application screenshots

## Documentation Structure

### Requirements
Detailed functional and non-functional requirements with acceptance criteria.

### Architecture
System design, component interactions, and technology stack.

### Use Cases
User stories and use case scenarios with actors and flows.

### UML Diagrams
Visual representations of system design:
- Use case diagram
- Sequence diagrams
- Class diagram
- Activity diagrams

### Screenshots
Visual documentation of the application interface and features.

## Generating Documentation

Documentation should be kept up-to-date with code changes. Update relevant sections when:
- Adding new features
- Modifying existing functionality
- Changing architecture or design decisions
- Updating dependencies or technology stack
